# kitty_desktop

`kitty_desktop` 是基于 [kitty](https://sw.kovidgoyal.net/kitty/) 内核能力构建的桌面封装版本。目标是在保留 kitty 高性能终端能力的前提下，提供更易上手的桌面化体验（安装、配置、启动、会话管理与可视化入口）。

## 项目定位

- **内核不替换**：终端渲染、协议能力、性能特性均依赖 kitty 原生实现。
- **桌面封装层**：通过桌面应用外壳（GUI + 配置管理 + 启动器）降低使用门槛。
- **面向日常开发**：提供开箱可用的工作区、会话和配置管理能力。

## 目标功能（MVP）

1. **桌面启动器**
   - 一键启动 kitty 实例
   - 支持常用参数模板（默认 shell、工作目录、主题）
2. **配置中心**
   - 可视化编辑核心配置（字体、主题、快捷键）
   - 配置变更后同步生成/更新 kitty 配置文件
3. **会话管理**
   - 保存与恢复常用工作会话（窗口布局、标签页、启动命令）
4. **扩展入口**
   - 快速打开 kitty 文档、配置目录、日志目录

## 参考实现架构

- **Core Adapter（内核适配层）**
  - 负责 kitty 可执行程序探测、版本检查、启动参数拼装
- **Desktop Shell（桌面壳层）**
  - 提供 GUI、系统托盘、菜单入口、快捷操作
- **Config Service（配置服务）**
  - 维护 UI 配置模型与 kitty 配置文件的双向映射
- **Session Service（会话服务）**
  - 持久化会话模板，支持快速恢复开发场景

## 实现计划（Roadmap）

### Phase 0：基础调研与约束确认

- 梳理 kitty 配置语法与可动态加载能力
- 明确跨平台桌面封装技术栈（Linux/macOS/Windows）
- 定义配置文件目录、日志目录、缓存目录规范

### Phase 1：可运行原型（MVP）

- 完成 kitty 运行时探测与启动封装
- 提供最小桌面 UI（启动、设置、退出）
- 支持基础配置项编辑并落盘
- 支持会话保存与恢复（基础版本）

### Phase 2：体验增强

- 增加主题预览与快捷键冲突检测
- 增加多配置模板（开发/运维/远程）
- 增加启动失败诊断信息与日志可视化

### Phase 3：发布与维护

- 打包发布（平台安装包）
- 补充用户文档与开发者文档
- 建立版本兼容矩阵（kitty 版本 × 平台版本）

## 非目标（当前阶段）

- 不重写终端渲染引擎
- 不替代 kitty 原生生态插件机制
- 不在首版覆盖全部高级配置项

## 下一步建议

- 先产出 `Core Adapter` 的命令行 PoC，验证 kitty 探测与参数注入流程
- 并行定义配置 schema，确保 GUI 表单与 kitty 配置映射稳定
- 在 MVP 阶段聚焦“可启动 + 可配置 + 可恢复会话”三项核心闭环
